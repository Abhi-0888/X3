Module A — Drone-BIM Navigator

Progress & accuracy

Concept

An AI-powered drone (simulated in Twinmotion/UE5) performs a daily auto-scan of the construction site. It carries a model trained on the 3D dataset — the "Ground Truth." Each live frame is compared against the 3D prototype's exact coordinates using point cloud alignment or voxel-based comparison to detect structural deviations and measure daily progress.

Algorithm skeleton — step by step

1
Frame extraction

Pull every Nth frame from drone video feed using OpenCV cv2.VideoCapture. N=5 gives ~6 FPS at real-time, enough for structural comparison.

2
Reference image load

Load the "Master 3D Frame" — a rendered screenshot from UE5/Twinmotion at the same drone camera angle. This is the ground truth for pixel-level comparison.

3
Feature matching — ORB/SIFT

Apply ORB (fast) or SIFT (accurate) keypoint detector on both frames. Use BFMatcher or FLANN to find corresponding feature points between live frame and master frame.

orb = cv2.ORB_create()
kp1, des1 = orb.detectAndCompute(frame_live, None)
kp2, des2 = orb.detectAndCompute(frame_master, None)
matches = bf.match(des1, des2)
matches = sorted(matches, key=lambda x: x.distance)
4
Homography + coordinate mapping

Use top N matches to compute homography matrix H (real frame → model frame). Apply cv2.findHomography with RANSAC to filter outliers. Map pixel coordinates to world coordinates using the BIM JSON.

5
Pixel-diff deviation map

Warp live frame into master frame's perspective using H. Subtract: diff = cv2.absdiff(warped_live, master). Threshold diff — pixels above threshold highlighted RED on the output overlay. These are the anomaly zones.

6
Deviation alert logic

Calculate anomaly area as % of total frame. If anomaly_area > 5% threshold → trigger deviation alert with element ID from BIM JSON lookup. Calculate progress delta vs yesterday's baseline.

deviation_pct = (anomaly_pixels / total_pixels) * 100
if deviation_pct > THRESHOLD:
  alerts.append({"type":"DEVIATION","element":element_id,"delta":deviation_pct})

Inputs

Video
Drone flythrough .mp4 (Twinmotion export)
BIM JSON
Element IDs + x/y/z world coords + type labels
Master img
UE5 rendered screenshot at drone angle

Outputs

Overlay
Live frame with red anomaly zones drawn
Alert log
JSON list of {element, deviation_pct, timestamp}
Progress
% wall built today vs yesterday baseline

Module B — 360° Guardian

Safety & buffer zones

Concept

A multi-camera array (Front, Back, Top, Left, Right) provides a spherical safety net across the entire site. Two sub-systems run in parallel: a YOLOv8/v11 PPE detector that checks every visible worker's compliance, and a dynamic buffer zone engine that defines danger perimeters around hazardous equipment and edges. Violations from either sub-system fire instant dashboard alarms.

Algorithm skeleton — Sub-system 1: PPE detection

1
Multi-stream capture

Open 5 video streams simultaneously using OpenCV. In simulation: 5 Twinmotion videos (front/back/top/left/right angles). In production: 5 RTSP camera streams.

caps = [cv2.VideoCapture(f"cam_{angle}.mp4") for angle in ["front","back","top","left","right"]]
2
YOLOv8 inference per frame

Run pre-trained YOLOv8-PPE model (weights from Roboflow or GitHub) on each frame from each camera. The model outputs class labels: helmet, no_helmet, vest, no_vest, person.

model = YOLO("ppe_yolov8.pt")
results = model(frame, conf=0.5)
for box in results[0].boxes:
  label = model.names[int(box.cls)]
  conf = float(box.conf)
3
Per-worker compliance check

For each detected person, check if both helmet AND vest detections overlap their bounding box. If either is missing → mark as NON-COMPLIANT.

4
Safety score calculation
safety_score = (compliant_count / total_workers) * 100
dashboard.update("safety_score", safety_score)

Algorithm skeleton — Sub-system 2: Dynamic buffer zones

1
Define danger perimeters

Pre-define danger zones as pixel-coordinate polygons or rectangles in each camera view. These represent areas around cranes, open edges, excavations, or live machinery.

DANGER_ZONES = {
  "crane_zone": [(120,80), (320,80), (320,350), (120,350)],
  "open_edge": [(0,400), (680,400), (680,480), (0,480)]
}
2
Worker centroid extraction

Calculate the centroid (cx, cy) of each worker's bounding box from YOLO output: cx = (x1+x2)/2, cy = (y1+y2)/2.

3
Point-in-polygon check

Use cv2.pointPolygonTest to check if worker centroid is inside any danger zone polygon. If yes AND worker is not in the authorized list → fire alarm.

for zone_name, polygon in DANGER_ZONES.items():
  dist = cv2.pointPolygonTest(np.array(polygon), (cx,cy), False)
  if dist >= 0:
    trigger_alarm(worker_id, zone_name)
4
Visual overlay

Draw danger zone rectangles in semi-transparent red on the live feed. Draw GREEN bounding box around compliant workers. Draw RED around non-compliant workers. Label each worker with ID.

Zone config
Danger zone polygon coordinates per camera

Outputs

Module C — Activity Analyst

Labor efficiency

Concept

A pose estimation engine quantifies productive movement by tracking 33 skeletal landmark joints per worker per frame using MediaPipe Pose. It distinguishes active labor (hammering, lifting, carrying) from static presence (standing still, idle). Individual and team efficiency scores are computed continuously and displayed on the dashboard.

Algorithm skeleton — step by step

1
MediaPipe Pose init

Initialize MediaPipe Pose model. Set min_detection_confidence=0.7. The model returns 33 landmarks per person: nose, shoulders, elbows, wrists, hips, knees, ankles.

import mediapipe as mp
mp_pose = mp.solutions.pose
pose = mp_pose.Pose(min_detection_confidence=0.7,
                  min_tracking_confidence=0.5)
2
Landmark extraction per frame

Convert frame to RGB. Run pose inference. Extract key joint positions — focus on wrists (L/R), elbows (L/R), and hips (L/R) as primary labor-activity indicators.

results = pose.process(rgb_frame)
if results.pose_landmarks:
  landmarks = results.pose_landmarks.landmark
  wrist_L = (landmarks[15].x, landmarks[15].y)
  wrist_R = (landmarks[16].x, landmarks[16].y)
3
Movement delta calculation

Per frame, calculate Euclidean displacement of each tracked joint vs previous frame. Sum all joint deltas to get a "movement score" for that frame.

def joint_delta(prev, curr):
  return math.sqrt((curr[0]-prev[0])**2 + (curr[1]-prev[1])**2)

movement_score = sum(joint_delta(prev[j], curr[j]) for j in KEY_JOINTS)
4
Active vs idle classification

Apply rolling window (30 frames ≈ 1 second). If avg movement_score > ACTIVE_THRESHOLD → state = "Active". If score < IDLE_THRESHOLD for >300 consecutive seconds → state = "Idle" → flag worker on dashboard.

ACTIVE_THRESHOLD = 0.015
IDLE_THRESHOLD = 0.003
IDLE_TIMEOUT_SEC = 300

if rolling_avg > ACTIVE_THRESHOLD:
  state = "Active"; active_frames += 1
elif idle_duration > IDLE_TIMEOUT_SEC:
  state = "Idle"; flag_worker(worker_id)
5
Efficiency score

Calculate per-worker efficiency as active ratio. Team efficiency = average of all individual scores. Feed both to dashboard in real time.

efficiency = (active_frames / total_frames) * 100
team_efficiency = sum(w.efficiency for w in workers) / len(workers)
6
Skeleton overlay

Draw MediaPipe skeleton on live frame using mp.solutions.drawing_utils. Color joints GREEN for active workers, AMBER for low-activity workers, RED for flagged idle workers.

Inputs

Video
Any camera feed with visible workers
Config
Activity thresholds, idle timeout seconds
Worker IDs
Tracked via YOLO detection + simple tracker

Outputs

Per worker
State (Active/Idle), efficiency %, idle duration
Team
Team efficiency %, active count, idle count
Overlay
Color-coded skeleton on each tracked worker
